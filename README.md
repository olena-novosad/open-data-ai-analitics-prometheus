# Monitoring Setup for Azure Docker Project

## Опис проєкту

У цьому проєкті реалізовано підсистему моніторингу для контейнеризованого застосунку, розгорнутого в Microsoft Azure.  
Моніторинг побудовано на основі **Prometheus**, **Grafana**, **Node Exporter** та **cAdvisor**.

**Метою є збір і візуалізація метрик із:**
- віртуальної машини;
- Docker-контейнерів;
- веб-застосунку.

**Інфраструктура створюється автоматично за допомогою Terraform, включаючи:**
- віртуальну машину (Linux VM)
- мережеву інфраструктуру (VNet, Subnet, Public IP, NSG)
- середовище для запуску контейнерів (Docker)

**Після створення VM за допомогою cloud-init автоматично:**
- встановлюється Docker та Docker Compose
- завантажується проєкт
- запускається аналітичний пайплайн + збір метрик через Docker Compose

## Структура моніторингу

```text
monitoring/
├── grafana/
├── prometheus/
│   └── prometheus.yml
└── docker-compose.monitoring.yml
```

- `prometheus/prometheus.yml` — конфігурація Prometheus для збору метрик;
- `grafana/` — каталог для службових даних або додаткового налаштування Grafana;
- `docker-compose.monitoring.yml` — файл для запуску сервісів моніторингу.

---

## Як розгортається моніторинг

Моніторинг інтегровано в існуюче розгортання в Azure.  
Було оновлено Terraform та cloud-init, щоб разом з основним застосунком автоматично запускались сервіси моніторингу.

Після внесення змін інфраструктура оновлюється стандартними командами Terraform:
```bash
terraform init
terraform validate
terraform plan
terraform apply
```
Після розгортання VM автоматично піднімаються:
- основний застосунок;
- стек моніторингу (Prometheus, Grafana, Node Exporter, cAdvisor).

## Які сервіси збирають метрики

У підсистемі моніторингу використовуються такі сервіси:

### Prometheus
Центральний сервіс збору метрик, який опитує інші сервіси та зберігає дані.

### Node Exporter
Node Exporter збирає системні метрики Linux-вузла, зокрема:
* завантаження CPU;
* використання оперативної пам’яті;
* файлові системи;
* мережеву активність;
* дискову активність.

### cAdvisor
cAdvisor збирає метрики Docker-контейнерів, зокрема:
* використання CPU контейнерами;
* використання пам’яті;
* мережевий трафік;
* стан контейнерів.

### Web Application
Веб-застосунок експортує метрики через endpoint `/metrics`.

## Які порти відкриті

- **3000** — Grafana (доступ через браузер)
- **9090** — Prometheus (для перевірки)
- **9100** — Node Exporter (внутрішній)
- **8080** — cAdvisor (внутрішній)
- **8000** — веб-застосунок (/metrics)

---

## Як відкрити Grafana

Після розгортання Grafana доступна через браузер за адресою: http://PUBLIC_IP:3000
, де PUBLIC_IP — публічна IP-адреса Azure VM.
---

## Підключення Prometheus у Grafana

Після входу в Grafana потрібно:
1. Перейти в **Data Sources**
2. Додати нове джерело
3. Обрати **Prometheus**
4. Вказати URL: http://prometheus:9090
5. Натиснути **Save & Test**

## Які панелі створено

У Grafana було використано як готові шаблони, так і власноруч створений дашборд.

### Node Exporter Dashboard

Було імпортовано готовий дашборд для Node Exporter, який відображає:

* завантаження CPU;
* використання RAM;
* мережевий трафік;
* використання диска;
* загальний стан Linux-вузла.

### cAdvisor Dashboard
Було імпортовано готовий дашборд для cAdvisor, який відображає:

* використання CPU контейнерами;
* використання пам’яті контейнерами;
* мережевий трафік контейнерів;
* інформацію про запущені контейнери.


### Власний дашборд
Було створено власний dashboard із такими метриками:
- VM CPU Usage
- VM RAM Usage
- Web Service CPU Load
- Кількість запитів
- Throughput (запити/сек)
- Загальна кількість запитів
- Average Latency
