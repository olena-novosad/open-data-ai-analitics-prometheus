# Cloud Deployment of Nuclear Safety Data Pipeline (Terraform + Docker)

## Опис проєкту

Цей проєкт реалізує контейнеризований аналітичний пайплайн для обробки відкритих державних даних з використанням інфраструктури, розгорнутої в Microsoft Azure за допомогою Terraform.

**Метою проєкту є:**
- аналіз екологічної та радіаційної обстановки в районах розташування АЕС України
- виявлення закономірностей у розподілі радіонуклідів
- перевірка дотримання нормативних значень безпеки

**Інфраструктура створюється автоматично за допомогою Terraform, включаючи:**
- віртуальну машину (Linux VM)
- мережеву інфраструктуру (VNet, Subnet, Public IP, NSG)
- середовище для запуску контейнерів (Docker)

**Після створення VM за допомогою cloud-init автоматично:**
- встановлюється Docker та Docker Compose
- завантажується проєкт
- запускається аналітичний пайплайн через Docker Compose

Таким чином забезпечується повністю автоматизоване розгортання та запуск застосунку в хмарі.

## Структура проєкту

Проєкт складається з модулів, які розгортаються у вигляді контейнерів на віртуальній машині Azure за допомогою Docker Compose:

- **data_load** — завантаження даних у PostgreSQL  
- **data_quality_analysis** — перевірка якості даних  
- **data_research** — аналітичні дослідження та перевірка гіпотез  
- **visualization** — генерація графіків  
- **web** — Flask веб-інтерфейс  

Додаткові директорії:

- **data** — вхідні дані (Excel)  
- **reports** — JSON-звіти (результати аналізу)  
- **plots** — згенеровані графіки  

Конфігураційні файли:

- **docker/docker-compose.yaml** — Docker Compose конфігурація для запуску сервісів  
- **docker/.env** — змінні середовища для контейнерів  
- **infra/terraform/** — Terraform конфігурація для розгортання інфраструктури  
- **README.md** — документація проєкту  

Структура організована за принципом розділення відповідальностей: кожен модуль відповідає за окремий етап обробки, аналізу та візуалізації даних.

## 🛠 Покрокова інструкція

### 1. Як відкрити та налаштувати Cloud Shell

1. Увійдіть у **Azure Portal**  
2. Натисніть іконку **Cloud Shell (>_)** у верхній панелі  
3. Оберіть середовище **Bash**

>  **Важливо:** Якщо це перший запуск, система запропонує створити сховище (**Mount Storage**).  
> Натисніть **Create storage**, щоб файли зберігалися після перезавантаження.
> 
> (цей крок не обов'язковий, але бажаний)

---

### 2. Підготовка та виконання Terraform

Перед запуском необхідно підготувати та завантажити файли конфігурації Terraform:

- `main.tf`  
  Основний файл інфраструктури. Містить опис усіх ресурсів Azure, які будуть створені:
  - Resource Group  
  - Virtual Network і Subnet  
  - Public IP  
  - Network Security Group (з правилами доступу)  
  - Network Interface  
  - Linux Virtual Machine  
  Також тут визначається передача `cloud-init` сценарію у VM через параметр `custom_data`.


- `variables.tf`  
  Файл зі змінними, що дозволяє гнучко налаштовувати конфігурацію без зміни основного коду.  
  Містить такі параметри, як:
  - назва Resource Group  
  - регіон розгортання  
  - ім’я віртуальної машини  
  - ім’я користувача  
  - URL GitHub-репозиторію з проєктом  


- `outputs.tf`  
  Файл, який визначає значення, що виводяться після виконання `terraform apply`.  
  Використовується для зручного отримання:
  - публічної IP-адреси VM  
  - URL веб-застосунку  


- `cloud-init.yaml`  
  Скрипт автоматичної конфігурації Linux VM, який виконується при першому запуску.  
  Забезпечує:
  - встановлення Docker  
  - встановлення Docker Compose  
  - клонування проєкту з GitHub  
  - створення конфігураційного файлу `.env`  
  - запуск застосунку через `docker compose up -d`  

### Організація файлів у Cloud Shell

Створіть необхідну структуру каталогів:
```bash
mkdir -p ~/project/infra/terraform
```

#### Завантаження файлів
У Cloud Shell:
1. Натисніть **Manage files**  
2. Оберіть **Upload**
3. Завантажте всі файли проєкту


Перемістіть файли конфігурації у створену директорію:
```bash
mv *.tf cloud-init.yaml ~/project/infra/terraform/
```
Перейдіть у каталог Terraform:
```bash
cd ~/project/infra/terraform
```
---

#### Запуск Terraform

```bash
# Ініціалізація провайдерів
terraform init

#Автоматичне форматування Terraform-файли відповідно до стандартів.
terraform fmt  

# Перевірка конфігурації
terraform validate

# Перегляд плану
terraform plan

# Розгортання інфраструктури
terraform apply 
```
---
### 3. Як перевірити результат

Після повідомлення **Apply complete!** виконайте перевірку.

#### В Azure Portal
- Перейдіть у **Resource Groups**
- Знайдіть свою групу
- Переконайтеся, що всі ресурси мають статус **Succeeded**

#### Перевірка веб-сервісу
Скопіюйте IP з Terraform:

```bash
terraform output public_ip_address
```

Відкрийте у браузері:

```bash
http://<ваш_ip_адреса>
```

---

### 4. Як виконати terraform destroy

Після завершення роботи обов’язково видаліть ресурси, щоб не витрачати кредит.

```bash
terraform destroy 
```

Дочекайтесь повідомлення:

**Destroy complete!**
